# AgToosa Review Rules

When running agtoosa-review or any code review, follow `Docs/AgToosa_Review.md` precisely.

## Key constraints

- Apply all five review personas: CEO, Security Engineer, Performance Engineer, DX Reviewer, UX/Accessibility Reviewer.
- Every BLOCKER must be resolved before the story can ship.
- Document all findings; update Linear with review status.
- Do not approve a story that has unresolved BLOCKERs.
